Initializing Lightning Network Daemon (LND)

In the first steps guide we created and configured the necessary password to unlock the LND service wallet, which was saved in the walletunlock file. Now we are going to create the wallet.

2.1 Creating the Wallet

Run the following:

cat app-data/lnd/walletunlock

This returns the password created earlier:

+6Mn31qVwLC-

Copy it and run:

docker exec -it node_lnd lncli create

This takes us to the wallet creation wizard.

Image 1: Wallet encryption key.

  1. Enter the password from the walletunlock file twice.

Image 2: Creating a new wallet.

  1. Enter n to create a new wallet.

Image 3: Creating a passphrase for the wallet.

  1. Press Enter to skip the passphrase.

Image 4: LND wallet seed phrase.

  1. Copy the seed phrase in a safe place away from the internet.
Note: This seed does not comply with the BIP39 standard, so if you try to restore it in a wallet like Sparrow, Electrum, or another, you will not see the on-chain funds.

Now we are going to edit the lnd.conf file and look for the line wallet-unlock-file

nano app-data/lnd/lnd.conf

Image 5: lnd.conf file parameter wallet-unlock-file.

  1. Remove the # symbol from in front of the line wallet-unlock-file.

Save and exit the file ctrl+s and ctrl+x

2.2 Acquiring Liquidity (Inbound Channels)

To receive payments in your Mint or LNbits, your node needs inbound liquidity. Being a new node, it has no open channels, so you must "rent" or buy initial liquidity through specialized markets.

2.2.1 Recommended Liquidity Services

  • LNBig: Allows requesting direct channels to your node. Ideal for fast liquidity and high capacity.
  • Amboss Magma: Marketplace where you can buy inbound liquidity channels. Allows filtering by reputation, duration, and cost.
  • Zeus LSP: Liquidity provider integrated with Zeus wallet, allows opening channels easily.
  • LN Server: Channel opening service with different capacity options.

To use these services, you will need the node URI, which has the format:

pubkey@ip:9735 or pubkey@address.onion:9735

2.2.2 Getting the LND Node URI Address

Run:

docker exec -it node_lnd lncli getinfo

Image 6: Result of the lncli getinfo command.

    1. Public key of the LND node.
    1. Node URI address both on Clearnet and Tor; either is valid for liquidity providers.
