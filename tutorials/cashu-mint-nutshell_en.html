Cashu Mint (Nutshell) Configuration

The Cashu Mint is the core component that issues and redeems ecash tokens. In this tutorial we will configure it to work with LNbits as the Lightning backend and manage its keysets.

6.1 Connecting LNbits as the Lightning Backend

Access the Mint configuration file:

nano app-data/cashu/.env

Look for the line MINT_BACKEND_BOLT11_LNURL_ENDPOINT and configure it with your LNbits instance:

MINT_BACKEND_BOLT11_LNURL_ENDPOINT=https://lnbits.cashu4community.xyz/lnurlp/api/v1/lnurlp/link/

Then look for MINT_LNBITS_ENDPOINT and set it to your LNbits API URL:

MINT_LNBITS_ENDPOINT=https://lnbits.cashu4community.xyz/api/v1

Finally, set MINT_LNBITS_KEY with an admin key from LNbits:

MINT_LNBITS_KEY=55739ee04e53437aba54fa8c9e4175b8

Image 1: LNbits Cashu Mint Wallet.

  1. Create a new wallet in LNbits for the Cashu Mint.
  2. Copy the Admin Key from the wallet.

Image 2: LNbits Cashu Wallet Admin Key.

  1. Paste the Admin Key in the MINT_LNBITS_KEY field in the .env file.

Save and exit the file ctrl+s and ctrl+x

6.2 Generating SSL Certificates for RPC Connection

SSL certificates are necessary to connect via RPC to the Mint and execute remote commands.

Go to the certs directory:

cd app-data/cashu/certs

Execute the certificate generation script:

./generate_certificates.sh

This script will generate all necessary certificates. We can verify by running:

ls .
ca_cert.pem  ca_private.pem  client_cert.pem  client_private.pem  generate_certificates.sh  server_cert.pem  server_private.pem

Image 3: Certificate files generated.

6.3 Creating a New Keyset from the CLI

We will use mint-cli to rotate keysets and define the fee. We will do this for each unit in the mint (sats, usd).

E.g.: if we wanted to change the fee from 100 ppk (per-proof-of-knowledge) for sat (Bitcoin) to 50 ppk, we would execute the following inside the cashu container:

docker exec -it app_cashu bash

Once inside the container, execute:

poetry run mint-cli -h cashu next-keyset sat 50
Note: The -h argument accompanying the mint-cli command is necessary because in Docker environments you cannot access 127.0.0.1 directly, which is why we use cashu as it is the service name declared in the docker compose.

This would return something like:

final_expiry = None
New keyset successfully created:
keyset.id = '01e5bac64190f22f2959558b15cbab9f06798958f253a59fbe4687a9d875e3c720'
keyset.unit = 'sat'
keyset.max_order = 64
keyset.input_fee_ppk = 50
0

If we now execute again the command:

curl https://mint.cashu4community.xyz/v1/keysets

It returns the pair of keysets: the old and the new.

6.4 USD Stable Support with Blink in the Mint

In version 0.20.0 of Nutshell, USD support via Blink as a funding source was added. In the following steps we will explain what you need to do both in Blink and in the Cashu Mint.

As a requirement, you must have an account in the Blink Lightning wallet; otherwise we cannot continue. If you don't have an account, download the application for Android from Google Play Store here or for iOS from the App Store here. Having the wallet registration, access https://dashboard.blink.sv

After logging in, you will see the account administration panel showing the two wallets (sats and usd) and several other options. The one we are interested in is API Keys.

Image 4: Blink Administration Panel - API Keys.

  1. After selecting API Keys in the options menu, you will see the following screen where we can create a new API key.

Image 5: Creating new API key.

  1. Click the button with the + symbol.

A window will appear where we will enter a name to identify the use of the key, the lifetime of the API key, and the operations we will allow with it.

Image 6: API key parameters.

  1. Enter the name of the API key, something that suggests the use it will have.
  2. API key expiration time: select Never.
  3. In the operations that can be done with the API, select all of them.
  4. Click the Create button to create the key.

After creating the key, a window appears where we can copy the API keys. It is recommended to back them up in a safe place. For our case, we are only interested in the one related to USD; but if you want to use Blink as a funding account for the mint in both USD and BTC.

Image 7: API keys to connect to Blink BTC and USD accounts.

  1. Copy the key in a safe place (this will be the one we use to connect the Cashu Mint to Blink).
  2. Click the Close button.

This way we have created the key we will use in the Mint. Now we return to the terminal in our VPS. We are in the cashu-deploy directory and will edit the .env file.

nano app-data/cashu/.env

Look for the line that has MINT_DERIVATION_PATH=:

MINT_DERIVATION_PATH="m/0'/0'/0"

Go to the beginning of the line and insert the # symbol, leaving it like this:

# MINT_DERIVATION_PATH="m/0'/0'/0"

Then look for the line MINT_DERIVATION_PATH_LIST=:

# MINT_DERIVATION_PATH_LIST=["m/0'/0'/0'", "m/0'/0'/1'", "m/0'/1'/0'", "m/0'/2'/0'"]

Remove the # symbol and the values between double quotes m/0'/0'/1' and m/0'/1'/0', leaving it like this:

MINT_DERIVATION_PATH_LIST=["m/0'/0'/0'", "m/0'/2'/0'"]
Note: You must be careful to remove a comma or delete a quote because it will cause an error after restarting the mint service.

Now look for the line MINT_BACKEND_BOLT11_USD=FakeWallet and remove the # symbol and where it says FakeWallet put BlinkWallet, leaving it like this:

MINT_BACKEND_BOLT11_USD=BlinkWallet

Finally, look for the Blink connection:

# MINT_BLINK_KEY=blink_abcdefgh

Remove the # symbol and paste the key we created in the Blink panel earlier, leaving something like this:

MINT_BLINK_KEY=blink_7Sz978fgdFGjhhT6u8g1tDvruhdeiuyidfdZ1fnDFcPSXKag5uHuip2S1yPP2wHG

Save and exit the file ctrl+s and ctrl+x

Restart the Cashu mint docker container.

docker-compose restart cashu

From here we will be able to select USD as a method to send and receive payments in wallets like cashu.me or elcaju.me.
